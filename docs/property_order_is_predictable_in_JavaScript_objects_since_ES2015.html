【翻译】从 ES2015 开始，JavaScript 对象属性顺序变得可以预测

常见的误解 – "无法保证 JavaScript 属性的顺序"

在你几年前刚开始编写 JavaScript 时，可能已经听说过 JS 对象中的属性顺序是不可预测的。虽然我从来没有碰到过奇怪的和不寻常的属性顺序，但是我仍然遵循"从不依赖属性顺序"的准则。

内置的 ownPropertyKeys 方法

现在的实际情况是，从 ES2015 开始，除去按时间顺序这种特殊情况，其他一些方法是基于定义顺序这种特定规则。而对象中属性的顺序，取决于所包含属性的类型和它的值。

查看规范，这些规则被定义在 内置 "ownPropertyKeys" 方法 中。例如，Object.getOwnPropertyNames 和 Reflect.ownKeys 这两个新方法内部都使用了此方法。

有趣的是，规范发生过一些变化，例如，从 ES5 到 ES6 的 Object.keys。ES6 规范中定义 Object.keys 同样依赖于 ownPropertyKeys，这使得它在当今的浏览器中也同样是可预测的。

但是，这也意味着你必须谨慎使用此方法，并且不应依赖于使用 Object.keys 时提供的可预测顺序，因为结果 可能会因浏览器的实现 而有所不同。

但是我们不妨深入理论：看一下 ownPropertyKeys 方法实现中定义的属性顺序。

1. 整数索引

所有属性为整数索引的情况，会排在整个对象属性顺序的首位，并按照数字排序。

const objWithIndices = {
  23: 23,
  '1': 1,
  1000: 1000
};

console.log(Reflect.ownKeys(objWithIndices));
// [1, 23, 1000]
// ☝️ 按照数字顺序

2. 字符串（但不是整数字符串）

不属于整数索引并且不是 Symbol 类型的属性紧随其后，并按照时间顺序排列。

const objWithStrings = {
  'bar': 'bar',
  '01': '01'
};

objWithStrings.last = 'last';
objWithStrings['veryLast'] = 'veryLast';

console.log(Reflect.ownKeys(objWithStrings));
// ['bar', '01', 'last', 'veryLast']
// ☝️ 按照时间顺序

3. Symbol

最后，Symbol 也按照时间顺序排列。

const objWithSymbols = {
  [Symbol('first')]: 'first',
  [Symbol('second')]: 'second'
};

objWithSymbols[Symbol('last')] = 'last';

console.log(Reflect.ownKeys(objWithSymbols));
// [Symbol(first), Symbol(second), Symbol(last)]
// ☝️ 按照时间顺序

总结

结合上面这些规则，你会发现整数始终位于对象属性的"首位"，其后是字符串和 Symbol。此外，我们可以控制字符串和 Symbol 属性的顺序，因为它们是按时间顺序排列的！

const obj = {
  '2': 'integer: 2',
  'foo': 'string: foo',
  '01': 'string: 01',
  1: 'integer: 1',
  [Symbol('first')]: 'symbol: first'
};

obj['0'] = '0';
obj[Symbol('last')] = 'symbol: last';
obj['veryLast'] = 'string: very last';

console.log(Reflect.ownKeys(obj));
// [ "0", "1", "2", "foo", "01", "veryLast", Symbol(first), Symbol(last) ]
// -> 1. 整数类型，按照数字顺序
// -> 2. 字符串类型，按照时间顺序
// -> 3. Symbol 类型，按照时间顺序

校正：正如 Malgosia Stepniak 指出的那样，"own property order(自有属性顺序)" 仅在现代浏览器中得到完全支持，而在其他浏览器中不支持，例如 IE。
